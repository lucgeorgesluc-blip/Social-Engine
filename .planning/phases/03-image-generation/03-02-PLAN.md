    **Implementation — add to autopilot/pipeline/run.js:**

    1. Add an exported function `stripHeroImage` near the top of the file (after imports, before `main()`):

    ```javascript
    /**
     * Remove the hero <img> tag from article HTML when image generation fails.
     * Matches the specific hero image pattern: class="w-full h-auto object-cover".
     * Does NOT remove other img tags (avatars, icons, etc.).
     *
     * @param {string} html - Article HTML string
     * @returns {string} HTML with hero img tag removed (or unchanged if no match)
     */
    export function stripHeroImage(html) {
      // Match <img that contains class="w-full h-auto object-cover" through the closing >
      // The tag spans multiple lines in generated articles
      return html.replace(
        /<img\b[^>]*class="w-full h-auto object-cover"[^>]*>/gs,
        ''
      );
    }
    ```

    The regex explanation:
    - `<img\b` — matches `<img` word boundary (won't match `<imgs` or `<image>`)
    - `[^>]*` — matches any attributes before class (like src, alt)
    - `class="w-full h-auto object-cover"` — the unique hero image class combination
    - `[^>]*>` — matches any remaining attributes after class through closing `>`
    - `s` flag — makes `.` and `[^>]` work across newlines (dotAll mode)
    - `g` flag — in case of duplicates (defensive)
