    **Implementation (autopilot/pipeline/image-generator.js):**

    Module exports: `generateImage` (async function) and `buildImagePrompt` (for testability).

    Function signature:
    ```javascript
    export async function generateImage({ slug, title, primaryKeyword, basePath, dryRun, _aiClient })
    ```

    The `_aiClient` parameter enables dependency injection for tests. Production callers omit it; the function creates `new GoogleGenAI({ apiKey })` internally.

    Implementation details:
    1. If `dryRun` is true: log info, return `{ success: false, reason: 'dry-run' }`
    2. Check `process.env.GOOGLE_AI_API_KEY` — if missing, return `{ success: false, reason: 'GOOGLE_AI_API_KEY not set' }`
    3. Create AI client: `const ai = _aiClient || new GoogleGenAI({ apiKey })`
    4. Read model name: `readFileSync(join(import.meta.dirname, '..', 'config', 'gemini-model.txt'), 'utf8').trim()` — yields `models/gemini-2.5-flash-image`
    5. Call `ai.models.generateContent({ model: MODEL_NAME, contents: buildImagePrompt(title, primaryKeyword), config: { responseModalities: ['IMAGE'], imageConfig: { aspectRatio: '16:9' } } })`
    6. Iterate `response.candidates[0].content.parts` to find the part with `inlineData` (not always index 0 — Pitfall 3)
    7. `Buffer.from(part.inlineData.data, 'base64')` to get PNG buffer
    8. `mkdirSync(outputDir, { recursive: true })` where outputDir = `join(basePath, 'assets', 'images', 'blog')`
    9. `sharp(buffer).resize(800, 450, { fit: 'cover' }).webp({ quality: 85 }).toFile(outputPath)` where outputPath = `join(outputDir, slug + '.webp')`
    10. Check file size with `statSync(outputPath).size` — if >307200 (300KB), re-encode at quality 70
    11. Return `{ success: true, imagePath: outputPath }`
    12. Entire body wrapped in try/catch: on error, `logger.warn(...)` and return `{ success: false, reason: err.message }`

    ```javascript
        `No text, no watermarks, no human faces. Photo-realistic style.`;
    }
    ```
