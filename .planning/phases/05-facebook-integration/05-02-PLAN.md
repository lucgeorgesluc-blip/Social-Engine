**dashboard/lib/schema-fb.sql** — APPEND the post_metrics table (do not overwrite existing fb_tokens and fb_sync_state):

```sql
-- Post-level Facebook metrics (synced daily)
CREATE TABLE IF NOT EXISTS post_metrics (
  id SERIAL PRIMARY KEY,
  post_id TEXT REFERENCES posts(id),
  fb_post_id VARCHAR(100),
  reach INTEGER DEFAULT 0,
  impressions INTEGER DEFAULT 0,
  engagement INTEGER DEFAULT 0,
  clicks INTEGER DEFAULT 0,
  synced_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(post_id, synced_at::date)
);
```

Note: The UNIQUE constraint uses `synced_at::date` to allow one metrics snapshot per post per day.

**dashboard/lib/fb-comments.js** — Comment polling service (CJS):

```javascript
const { query } = require('./db');
const { getPageToken } = require('./fb-token');
```

Export `async function syncComments()`:
1. `const token = await getPageToken()` — if null, return `{synced: 0, error: 'no_token'}`
2. Get last poll timestamp: `SELECT value FROM fb_sync_state WHERE key = 'comments_last_poll'`. Default to 24 hours ago if no row.
3. Fetch all published posts with Facebook IDs: `SELECT id FROM posts WHERE platform = 'facebook' AND status = 'published'`
4. For each post, call Graph API: `https://graph.facebook.com/v21.0/${postId}/comments?fields=id,message,from,created_time&since=${lastPollTimestamp}&access_token=${token.access_token}`
   - Use Node 20 built-in `fetch()` — no npm package
   - Parse response JSON. If `error` field, log and continue to next post.
5. For each comment in response `data` array, upsert:
   ```sql
   INSERT INTO comments (id, post_id, date, platform, author_name, comment_text, response_status)
   VALUES ($1, $2, $3, 'facebook', $4, $5, 'pending')
   ON CONFLICT (id) DO NOTHING
   ```
   - `id` = Facebook comment ID (e.g., `"123456_789"`)
   - `post_id` = the dashboard post ID
   - `date` = parse `created_time` to DATE
   - `author_name` = `from.name` (may be null if permissions missing)
   - `comment_text` = `message`
6. Update sync state: `INSERT INTO fb_sync_state (key, value, updated_at) VALUES ('comments_last_poll', $1, NOW()) ON CONFLICT (key) DO UPDATE SET value = $1, updated_at = NOW()` with current ISO timestamp
7. Return `{synced: totalInserted, posts_checked: postCount}`
8. Wrap entire function in try/catch — log errors, return `{synced: 0, error: err.message}`

**dashboard/lib/fb-metrics.js** — Metrics sync service (CJS):

```javascript
const { query } = require('./db');
const { getPageToken } = require('./fb-token');
```

Export `async function syncMetrics()`:
1. `const token = await getPageToken()` — if null, return `{synced: 0, error: 'no_token'}`
2. Fetch published posts from last 7 days: `SELECT id FROM posts WHERE platform = 'facebook' AND status = 'published' AND published_date >= NOW() - INTERVAL '7 days'`
3. For each post, call Graph API: `https://graph.facebook.com/v21.0/${postId}/insights?metric=post_impressions,post_reach,post_engaged_users,post_clicks&access_token=${token.access_token}`
   - Use Node 20 built-in `fetch()`
   - Parse response. Extract values from `data` array where each metric has `values[0].value`
4. Upsert into post_metrics:
   ```sql
   INSERT INTO post_metrics (post_id, fb_post_id, reach, impressions, engagement, clicks)
   VALUES ($1, $2, $3, $4, $5, $6)
   ON CONFLICT (post_id, (synced_at::date)) DO UPDATE SET
     reach = $3, impressions = $4, engagement = $5, clicks = $6, synced_at = NOW()
   ```
5. Update sync state: key `'metrics_last_sync'` with current ISO timestamp
6. Return `{synced: metricsCount, posts_checked: postCount}`
7. Wrap in try/catch — log errors, return `{synced: 0, error: err.message}`

**dashboard/lib/fb-cron.js** — Cron job scheduler (CJS):

```javascript
const cron = require('node-cron');
const { syncComments } = require('./fb-comments');
const { syncMetrics } = require('./fb-metrics');
```

Export `function startCronJobs()`:

1. Comment polling — every 15 minutes:
   ```javascript
   cron.schedule('*/15 * * * *', async () => {
     try {
       const result = await syncComments();
       console.log('[cron] Comment sync:', result);
     } catch (err) {
       console.error('[cron] Comment sync failed:', err.message);
     }
   });
   ```

2. Metrics sync — daily at 6:00 AM (Paris time = UTC+2, so 4 AM UTC):
   ```javascript
   cron.schedule('0 4 * * *', async () => {
     try {
       const result = await syncMetrics();
       console.log('[cron] Metrics sync:', result);
     } catch (err) {
       console.error('[cron] Metrics sync failed:', err.message);
     }
   });
   ```

3. Log startup: `console.log('[cron] Facebook sync jobs started: comments (*/15 min), metrics (daily 6AM Paris)');`

Each job is wrapped in try/catch — errors are logged but never crash the server.

**dashboard/routes/fb.js** — ADD two manual sync routes (append to existing file, before `module.exports`):
