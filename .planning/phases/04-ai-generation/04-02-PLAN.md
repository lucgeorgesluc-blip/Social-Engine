**1. dashboard/views/ai.ejs** — Full AI generation page. Receives locals: `postTypes`, `usage`, `suggestions`, `generated`, `error`, `selectedType`, `context`.

Structure (all text in French, Tailwind classes, matching dashboard style):

```
&lt;div class="max-w-4xl mx-auto"&gt;
  &lt;h1 class="text-2xl font-bold text-secondary mb-6"&gt;Generation IA&lt;/h1&gt;

  &lt;!-- Usage counter banner --&gt;
  &lt;div&gt; showing "X / 100 generations ce mois"
    - If usage.count &gt;= usage.warning (50): yellow/amber bg with warning text "Attention : vous approchez de la limite mensuelle"
    - If usage.count &gt;= usage.limit (100): red bg with "Limite mensuelle atteinte (100). Reessayez le mois prochain."
    - Otherwise: green/gray bg, neutral text
  &lt;/div&gt;

  &lt;!-- Objection suggestions (if any) --&gt;
  &lt;% if (suggestions &amp;&amp; suggestions.length &gt; 0) { %&gt;
  Blue box with h3 "Suggestions basees sur les commentaires"
  Loop suggestions: link to /dashboard/ai?type=objection-buster&amp;context=encodeURIComponent(s.objection_type)
  Text: Ecrire sur "objection_type" (vu frequency fois)
  &lt;% } %&gt;

  &lt;!-- Generation form --&gt;
  form method="POST" action="/dashboard/ai/generate"
    - select name="type" with all postTypes as options, selectedType pre-selected
    - textarea name="context" id="context" rows="2" (shown/hidden by JS based on needsContext)
    - button type="submit" "Generer" — disabled if usage.count &gt;= usage.limit

  &lt;!-- Error display --&gt;
  Red box showing error text if error is truthy

  &lt;!-- Generated text + inline edit + save as draft --&gt;
  if generated is truthy:
    form method="POST" action="/dashboard/posts"
      textarea name="content" rows="12" pre-filled with generated text (editable by user)
      hidden inputs: type=selectedType, status=draft, platform=facebook, source=ai
      button "Sauvegarder comme brouillon" (submit)
      button "Copier le texte" (type=button, onclick: navigator.clipboard.writeText)
&lt;/div&gt;

&lt;!-- Inline JS for context field show/hide --&gt;
script: on type select change, find matching postType, show/hide context-group div,
update context-label text to pt.contextLabel. Use JSON.stringify(postTypes) for client-side data.
```

**3. dashboard/views/dashboard.ejs** — Add objection suggestion widget to homepage.

The dashboard route (GET /) must pass suggestions. Update `dashboard/routes/dashboard.js`:

Add at top: `const { getFrequentObjections } = require('../lib/ai');`

In the `GET /` handler, after the counts query, add:
```javascript
let suggestions = [];
try {
  suggestions = await getFrequentObjections();
} catch (e) {
  // Non-blocking — homepage still renders without suggestions
}
```
Pass `suggestions` to `res.render('dashboard', { ..., suggestions })`.

In `dashboard.ejs`, add after the existing health/counts cards a new section:
```html
<% if (typeof suggestions !== 'undefined' && suggestions.length > 0) { %>
<div class="bg-white rounded-lg shadow p-6 mt-6">
  <h2 class="text-lg font-semibold text-secondary mb-3">Suggestions de posts</h2>
  <p class="text-sm text-gray-600 mb-3">Ces objections reviennent souvent dans les commentaires :</p>
  <ul class="space-y-2">
    <% suggestions.forEach(s => { %>
    <li class="flex items-center justify-between">
      <span class="text-sm"><%= s.objection_type %> <span class="text-gray-400">(vu <%= s.frequency %> fois)</span></span>
      <a href="/dashboard/ai?type=objection-buster&context=<%= encodeURIComponent(s.objection_type) %>"
         class="text-xs bg-primary/10 text-primary px-3 py-1 rounded-full hover:bg-primary/20 transition">
        Generer un post
      </a>
    </li>
    <% }) %>
  </ul>
</div>
<% } %>
```
