**dashboard/lib/seed.js** — per research Pattern 5 and INFRA-03:

Read each YAML file from `../../.social-engine/data/` using `path.resolve(__dirname, '../../.social-engine/data/[filename].yaml')`.

For each YAML file, implement an async function:

1. `seedPosts()` — reads `posts.yaml`. The YAML structure has a `posts` array. For each post object, INSERT into posts table mapping fields: id, status, platform, type, hook, cta_type, objection_addressed, created_date, published_date, tags (JSON.stringify), metrics (JSON.stringify if present, else '{}'), is_draft=false, content=null. Use `ON CONFLICT (id) DO NOTHING`.

2. `seedDrafts()` — reads `posts-drafts.yaml`. Same table (posts), but set is_draft=true and map the content field. Use `ON CONFLICT (id) DO NOTHING`.

3. `seedComments()` — reads `comments.yaml`. Map: id, post_id, date, platform, author_name, full_name, classification, objection_type, comment_text, response_text, response_status, converted_to_dm, notes. Use `ON CONFLICT (id) DO NOTHING`. Note: post_id must reference an existing post — seed posts first.

4. `seedProspects()` — reads `dm-pipeline.yaml`. Map: id, source_comment_id, source_post_id, platform, prospect_name, full_name, date_first_contact, stage, messages (JSON.stringify), calendly_date, conversion_date, lost_reason, notes. Use `ON CONFLICT (id) DO NOTHING`.

5. `seedMetrics()` — reads `metrics-weekly.yaml`. Map: week, dates, posts_published, total_reach, total_impressions, total_likes, total_comments, total_shares, info_comments, dm_opened, calendly_booked, patients_converted, engagement_rate, best_post_id, worst_post_id, top_objection, learnings. Use `ON CONFLICT (week) DO NOTHING`.

Export a single `runSeed()` async function that calls them in order: posts first (parent), then drafts, then comments, then prospects, then metrics. Wrap each in try/catch — log errors with `console.error('[seed] Error seeding [table]:', err.message)` but do NOT throw (one table failing must not prevent others).

Return an object `{ posts: N, drafts: N, comments: N, prospects: N, metrics: N }` with inserted counts (use `RETURNING id` to count).

**Note on YAML structures:** Read each YAML file carefully. The top-level key varies by file (e.g., `posts:`, `comments:`, `prospects:`, `metrics:`). Adapt parsing to actual structure. If a file doesn't exist, log a warning and skip — do not crash.
