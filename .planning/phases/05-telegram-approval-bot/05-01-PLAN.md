    **Step 3: Modify `autopilot/pipeline/deploy-orchestrator.js`** -- add array-based functions while keeping existing ones for backward compat:

    Add 4 new exports (keep existing `writePending`, `readPending`, `triggerDeploy` intact):

    `readPendingArray({ _stateDir } = {})`:
    - If no file: return `[]`
    - Parse JSON. If `Array.isArray(data)` return data. Else return `[data]` (migration guard per Pitfall 2).

    `writePendingArray(items, { _stateDir } = {})`:
    - `mkdirSync` + `writeFileSync(join(stateDir, 'pending.json'), JSON.stringify(items, null, 2), 'utf8')`

    - Push item, call `writePendingArray`.

    `findPendingBySlug(slug, { _stateDir } = {})`:
    - `readPendingArray().find(i => i.slug === slug)`

    `removePendingBySlug(slug, { _stateDir } = {})`:
    - Filter out slug, call `writePendingArray` with filtered array.

    Import `MAX_PENDING_ARTICLES` from `../config/constants.js`.

  <action>
    Create `autopilot/telegram/bot.js` as ESM module with these exports:

    **`createBot(token, { _deps } = {})`** -- DI pattern for testing:
    - `const bot = new Telegraf(token)`
    - `_deps` allows injecting: `{ triggerDeploy, readPendingArray, findPendingBySlug, removePendingBySlug, writePendingArray, formatPreviewMessage }` -- defaults to real imports
    - Register handlers (described below)
    - Return `bot`

    **`buildPreviewKeyboard(slug)`**:
    ```javascript
    return Markup.inlineKeyboard([
      [
        Markup.button.callback('\u2705 Approuver et deployer', `approve:${slug}`),
        Markup.button.callback('\u270F\uFE0F Modifier', `edit:${slug}`),
      ],
    ]);
    ```

    **`sendPreview(bot, chatId, pendingItem, html)`**:
    - Call `formatPreviewMessage(pendingItem, html)` to get text
    - Call `bot.telegram.sendMessage(chatId, text, { ...buildPreviewKeyboard(pendingItem.slug) })`
    - Return the sent message (store `message_id` on pendingItem for potential future use)

    **Handler: `bot.action(/^approve:(.+)$/, async (ctx) => { ... })`**:
    - Guard: `if (ctx.from.id !== Number(process.env.TELEGRAM_CHAT_ID)) return;`
    - `await ctx.answerCbQuery('Deploiement en cours...')`
    - Extract slug: `ctx.match[1]`
    - Find item: `findPendingBySlug(slug)`. If not found, reply "Article non trouve" and return.
    - Call `triggerDeploy({ slug, _stateDir })` in try/catch
    - On success: `ctx.reply('\u2705 Article deploye : https://www.magnetiseuse-lacoste-corinne.fr/blog/${slug}.html')`
    - On error: `ctx.reply('\u274C Echec du deploiement : ${err.message}')`

    **Handler: `bot.action(/^edit:(.+)$/, async (ctx) => { ... })`**:
    - Guard: check chat ID
    - `await ctx.answerCbQuery()`
    - Extract slug, find item in pending array
    - If `item.edit_count >= MAX_EDIT_CYCLES`: send D-19 alert and call `removePendingBySlug`. Return.
    - Set `item.awaiting_edit = true`, save via `writePendingArray`
    - Reply: `"Qu'est-ce que vous voulez modifier pour '${item.title}' ?"`

    **Handler: `bot.on('text', async (ctx) => { ... })`**:
    - Guard: check chat ID
    - Read pending array, find item with `awaiting_edit === true`
    - If none: ignore (return silently)
    - Store edit text: `item.edit_history.push(ctx.message.text)`, `item.awaiting_edit = false`
    - Save array (edit-in-place call deferred to Plan 02 -- for now just save state and reply "Edit recu, traitement en cours...")
    - NOTE: The actual Claude edit-in-place call is wired in Plan 02. This task only sets up the state management.

    **`startBot(app, { _token, _deps } = {})`**:
    - `const token = _token || process.env.TELEGRAM_BOT_TOKEN`
    - If no token: log warning and return (graceful skip for dev without bot token)
    - `const bot = createBot(token, { _deps })`
    - If `process.env.NODE_ENV === 'production'`:
      - `const webhookUrl = process.env.RENDER_URL`
      - `app.use(await bot.createWebhook({ domain: webhookUrl, path: '/telegram/webhook' }))`
      - Log: `'Bot webhook registered at /telegram/webhook'`
    - Else:
      - `bot.launch()`
      - `process.once('SIGINT', () => bot.stop('SIGINT'))`
      - `process.once('SIGTERM', () => bot.stop('SIGTERM'))`
      - Log: `'Bot started in polling mode'`
    - Return `bot`

    **Tests in `autopilot/tests/telegram-bot.test.js`**:
    - Test `buildPreviewKeyboard` returns object with `reply_markup.inline_keyboard` containing "approve:" and "edit:" callback data
    - Test approve handler: mock `_deps.triggerDeploy` to resolve, verify it was called with correct slug
    - Test approve handler with deploy failure: mock triggerDeploy to reject, verify error message sent
    - Test edit handler: verify `awaiting_edit` set to true in pending array after edit callback
    - Test text handler with awaiting item: verify edit_history gets the message text, awaiting_edit reset to false
    - Test text handler without awaiting item: verify no state change
    - Test chat ID guard: verify handler returns without action for wrong chat ID

    For handler tests: create a mock context object with `{ from: { id }, match: [full, slug], answerCbQuery: stub, reply: stub }` and call handlers directly via the DI `_deps` pattern.
