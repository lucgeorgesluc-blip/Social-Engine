**1. Create `autopilot/config/audit-config.js`:**

```javascript
// Never-auto-apply issue codes -- these route to human review, never auto-apply
export const NEVER_AUTO_APPLY = [
  'AGGREGATE_RATING',       // Risk of Google manual action
  'CANONICAL_CHANGE',       // Page already has a canonical -- changing is risky
  'TITLE_CHANGE',           // H1/title text changes affect rankings
  'ROBOTS_CHANGE',          // robots meta tag changes affect crawling
  'MULTI_FILE_PATCH',       // Any patch touching more than one file
];

// Claude model for patch generation (same as article generation)
export const PATCH_MODEL = 'claude-sonnet-4-6';
export const PATCH_MAX_TOKENS = 4000;
```

**2. Create `autopilot/audit/patch-generator.js`:**

ESM module with these named exports:
- `isNeverAutoApply(issues)` -- pure function, returns `true` if any issue code is in NEVER_AUTO_APPLY list. Takes an array of issue objects `[{ code, severity, message }]`. Returns boolean.
- `buildPatchSystemPrompt()` -- returns the system prompt string. Content: "You are an SEO HTML patch generator for a French wellness website. Output ONLY valid HTML snippets. No commentary, no markdown fences. Rules: never hard-coded euros (use data-price=), never mention rTMS, use HealthAndBeautyBusiness (not MedicalBusiness) for schema @type, all JSON-LD must be valid JSON inside script tags, output ONLY the snippet to inject not the full page."
- `buildPatchUserPrompt(slug, issues, pageContext)` -- returns user prompt string. Includes: slug, JSON.stringify(issues), pageContext.headExcerpt, JSON.stringify(pageContext.schemaData). Instructs Claude to generate HTML snippet(s) to fix the listed issues. Output format: one `<script type="application/ld+json">` block per schema to add, or one `<link>` tag if adding canonical. Return ONLY valid HTML.
- `generatePatch(slug, issues, pageContext, { _client, _logCost } = {})` -- async function. Steps:
  1. Check `isNeverAutoApply(issues)` -- if true, log warning via pino and return `{ blocked: true, reason: 'never-auto-apply', snippet: null, usage: null }`
  2. Create client: `_client || new Anthropic()`
  3. Call `client.messages.stream({ model: PATCH_MODEL, max_tokens: PATCH_MAX_TOKENS, system: buildPatchSystemPrompt(), messages: [{ role: 'user', content: buildPatchUserPrompt(slug, issues, pageContext) }] })`
  4. Accumulate snippet from stream.on('text')
  5. `const finalMsg = await stream.finalMessage()`
  6. Log cost via `(_logCost || logCost)(finalMsg.usage, slug, logPath)` where logPath = `join(__dirname, '..', COST_LOG_PATH)`
  7. Return `{ blocked: false, snippet: snippet.trim(), usage: finalMsg.usage }`

DI pattern: `_client` for Anthropic mock, `_logCost` for cost logger mock. Follow exact same streaming pattern as `pipeline/generator.js`.

Import: `Anthropic from '@anthropic-ai/sdk'`, `pino`, `{ logCost } from '../pipeline/cost-logger.js'`, `{ COST_LOG_PATH } from '../config/constants.js'`, `{ NEVER_AUTO_APPLY, PATCH_MODEL, PATCH_MAX_TOKENS } from '../config/audit-config.js'`.

**3. Create `autopilot/audit/patch-validator.js`:**

ESM module with single named export `validatePatch(originalHtml, patchedHtml)`. Pure function, no I/O, no DI needed.

Uses `import { load } from 'cheerio'` with `{ decodeEntities: false }` option on all `load()` calls (RESEARCH Pitfall 1: preserve French accents).

8 checks returning `{ valid: boolean, failedChecks: string[] }`:

1. `HTML_PARSE_ERROR` -- try `load(patchedHtml)`, if throws push error and return early
2. `INVALID_JSONLD` -- for each `script[type="application/ld+json"]` in patched HTML, try `JSON.parse($(el).text())`. Push if any fails.
3. `CANONICAL_COUNT` -- `$('link[rel="canonical"]').length` must equal exactly 1. Push `CANONICAL_COUNT` if not 1.
4. `ALPINE_XDATA_REMOVED` -- count `[x-data]` elements in original vs patched. If patched count < original count, push.
5. `NOT_IDEMPOTENT` -- this check is a placeholder that returns a helper message. The actual idempotency is checked by the caller (apply flow) since it needs to re-run the snippet injection. In validatePatch itself, skip this check (document it as "caller responsibility" in JSDoc).
6. `DUPLICATE_SCHEMA_TYPE` -- collect all `@type` values from all JSON-LD blocks. If any `@type` appears more than once, push.
7. `WORD_COUNT_DECREASED` -- compare body text word count (split on whitespace, filter empty). If patched < original, push.
8. `DATA_PRICE_REMOVED` -- count `[data-price]` elements in original vs patched. If patched < original, push.

Return `{ valid: failedChecks.length === 0, failedChecks }`.
