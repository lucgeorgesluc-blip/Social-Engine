# Phase 1 — UI Design Contract

> Visual and interaction contract for the dashboard shell: login page + post-login layout with health status card.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS custom build) |
| Preset | not applicable |
| Component library | none |
| Icon library | none (Phase 1 uses text labels; icon library deferred to Phase 2) |
| Font | Inter (sans-serif) — self-hosted from `assets/fonts/inter-latin.woff2` |

**Source:** D-03 (stack), D-04 (Tailwind), tailwind.config.js (font tokens)

The dashboard requires its own Tailwind build in `dashboard/` with its own `tailwind.config.js` pointing to `dashboard/views/**/*.ejs`. The main site's pre-built `tailwind.css` must NOT be reused (class gap risk, see project memory).

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Input padding vertical, button gap to icon |
| md | 16px | Card inner padding, form field spacing |
| lg | 24px | Card outer margin, sidebar item spacing |
| xl | 32px | Login card padding, content area padding |
| 2xl | 48px | Login page vertical centering gap |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Touch targets on mobile sidebar hamburger button must be minimum 44px × 44px (INFRA-05 mobile usability).

| Heading (card title) | 18px | 600 | 1.3 |

## Color

Brand palette from `tailwind.config.js` (D-04).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F9FAFB (Tailwind gray-50) | Page background, content area background |
| Secondary (30%) | #2C5F4F (brand secondary) | Sidebar background, nav item active state |
| Accent (10%) | #DC512C (brand primary) | Primary CTA button, "Se connecter" button, active nav indicator |
| Surface | #FFFFFF | Login card, health status card, all card backgrounds |
| Destructive | #DC2626 (Tailwind red-600) | Error messages (wrong password, DB connection failed) |

Accent reserved for: "Se connecter" button fill, active sidebar nav item left border (4px), logout button hover state.

Secondary (dark green) used for: full sidebar background, nav item text (white on dark green), sidebar header text.

## Component Contracts

### Login Page

- Full-viewport centered layout: `min-h-screen flex items-center justify-center bg-gray-50`
- Card: `bg-white rounded-xl shadow-md p-8 w-full max-w-sm`
- Header block: "Social Dashboard" (Display, 24px/700, text-secondary) above "Corinne Lacoste" (Label, 12px/500, text-gray-500)
- Single `<input type="password">` field, placeholder "Mot de passe", full width
- Submit button: `bg-primary text-white w-full py-3 rounded-lg font-semibold text-sm hover:bg-opacity-90 transition` — label "Se connecter"
- Error state (wrong password): red text below input field, 12px, "Mot de passe incorrect. Réessayez."
- No username field (D-07)

### Sidebar Navigation

- Fixed left sidebar: 240px wide on desktop, full-height, `bg-secondary text-white`
- Collapsed to hamburger on mobile (≤375px): slide-in drawer overlay, hamburger button 44×44px touch target, top-left of screen
- Sidebar header: "Social Dashboard" text (14px/600, white) + "Corinne Lacoste" subtext (12px/400, white/70)
- Nav items (all visible from day 1 per D-09):
  - Accueil — enabled (active in Phase 1)
  - Posts — disabled, greyed text + "Bientôt" badge
  - Commentaires — disabled, greyed text + "Bientôt" badge
  - Pipeline — disabled, greyed text + "Bientôt" badge
  - Statistiques — disabled, greyed text + "Bientôt" badge
- Active item: left border 4px solid #DC512C (accent), white text, slightly lighter bg
- Disabled item: text-white/40, pointer-events-none, `<span class="ml-auto text-xs bg-white/10 text-white/50 px-2 py-0.5 rounded-full">Bientôt</span>`
- Logout button: bottom of sidebar, "Se déconnecter", text-white/70, hover:text-white, POST /logout

### Health Status Card (Homepage — D-10)

- Card: `bg-white rounded-xl shadow-sm border border-gray-100 p-6`
- Greeting: "Bienvenue, Benjamin" (Heading, 18px/600, text-secondary)
- DB status row: green dot (●) + "Base de données connectée" OR red dot + "Erreur de connexion"
- Seed data counts: 3 stat chips in a row — Posts (N), Commentaires (N), Prospects (N)
- Each stat chip: `bg-gray-50 rounded-lg px-4 py-3 text-center` — count (24px/700, text-secondary), label below (12px/400, text-gray-500)
- If DB error: chips show "—" and a full-width red notice below: "Impossible de charger les données. Vérifiez la connexion à la base de données."
